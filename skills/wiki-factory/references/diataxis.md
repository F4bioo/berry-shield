# 🗺️ Diátaxis Framework for Berry Shield

Follow this structure for all Wiki pages to ensure technical clarity and goal-oriented navigation.

## 1. Tutorials (Learning-oriented)
- **Goal**: Help the user start using a feature.
- **Tone**: Friendly but concise.
- **Form**: Step-by-step instructions.
- **Example**: "Your First Security Audit".

## 2. How-to Guides (Task-oriented)
- **Goal**: Help the user solve a specific problem.
- **Tone**: Direct and practical.
- **Form**: A series of steps for a known task.
- **Example**: "How to Redact Custom API Tokens".

## 3. Reference (Information-oriented)
- **Goal**: Describe the machinery.
- **Tone**: Purely technical, dry, and factual.
- **Form**: API tables, parameter lists, type definitions.
- **Example**: `utils/redaction.ts` documentation.

## 4. Explanation (Understanding-oriented)
- **Goal**: Explain why things are the way they are.
- **Tone**: Discursive and analytical.
- **Form**: Architectural diagrams (Mermaid), rationale, background.
- **Example**: "The Architecture of the Redaction Engine".
